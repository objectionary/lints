Object Has Data

Each object <o/> in XMIR must not have the data, unless itâ€™s base is org.eolang.bytes (or just bytes for short).

Incorrect:

<object>
  <o name="bar" base="foo">
    <o>A1-B2-C3-D4-E5</o>
  </o>
</object>

Correct:

<object>
  <o name="bar" base="org.eolang.bytes">
    <o>A1-B2-C3-D4-E5</o>
  </o>
</object>
