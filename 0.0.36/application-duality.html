Application Duality

In EO, it is not supported to have an application with and without binding at the same time, so it is prohibited in the XMIR too.

Incorrect:

<o base="man">
  <o base="string" as="name"/>
  <o base="number"/>
</o>

Correct:

<o base="man">
  <o base="string" as="name"/>
  <o base="number" as="age"/>
</o>

Or

<o base="man">
  <o base="string"/>
  <o base="number"/>
</o>
